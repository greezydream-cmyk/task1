Некоторые основные команды Git связаны с работой с изменениями, ветками и удалёнными репозиториями. Также существуют команды для настройки Git.

## Работа с изменениями

*git status* — проверка состояния репозитория. Показывает, какие файлы находятся в индексе и готовы к коммиту, какие файлы были изменены, но ещё не добавлены в индекс, какие новые файлы появились в проекте и пока не отслеживаются Git. 

*git add* — добавление файла или файлов в индекс, подготовка их для коммита. Можно добавить все файлы из директории — git add .. 

*git commit* — сохранение изменений в репозитории. Например, git commit -m "Сообщение коммита". 

*git diff* — просмотр изменений до коммита. Показывает разницу между последним коммитом и текущим состоянием репозитория. Можно добавить имя файла и сравнить его содержимое с последним коммитом, или использовать хеш коммита. 

*git reset* — отмена изменений в репозитории. Может изменять как индекс, так и рабочий каталог. Есть несколько режимов, которые определяют степень отката изменений: --soft, --mixed и --hard. 

## Ветви

*git branch* — создание новой ветки. Если не передать аргумент, команда выведет список всех локальных веток. 

*git checkout* — переключение на другую ветку. Можно передать опцию -b, чтобы создать новую ветку перед переключением. 

*git branch -d* — удаление ветки. 

*git merge* — слияние веток. Позволяет добавить изменения из одной ветки в другую, завершается появлением общего коммита для объединённых веток. 58
Удалённые репозитории

*git remote* — команда для управления связями между локальным репозиторием и другими версиями кода. Позволяет подключать, просматривать, переименовывать и ликвидировать удалённые репозитории.

*git remote add <имя>* — добавление нового репозитория.

*git remote remove <имя>* — удаление репозитория из конфигурации.

*git remote rename <старое> <новое>* — присвоение нового имени удалённому репозиторию.

*git remote set-url <имя>* — изменение URL удалённого репозитория.
 
## Параметры

Некоторые параметры, которые можно передавать командам Git:

*--all* — для команды *git add* — добавляет все новые и изменившиеся файлы. 

*--no-edit* — позволяет внести в коммит поправку без изменения сообщения коммита. 

*--verbose* — для команды *git pull* — отображает подробные сведения о загруженных файлах. 

*--force* — для команды *git push* — принудительная отправка изменений в удалённый репозиторий с перезаписью истории удалённого репозитория.
